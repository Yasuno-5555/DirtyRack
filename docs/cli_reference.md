# DirtyRack Forensic CLI Reference (Draft)

`dirtyrack-cli` is a command-line tool for deterministic audio rendering and forensic validation of patch identity.

## Basic Commands

### `dirtyrack render <PATCH_JSON> [OPTIONS]`
Renders the specified patch file offline and outputs a deterministic audio file.
- `--output <FILE>`: Output filename (.wav).
- `--length <SEC>`: Length of the audio to render.
- `--sample-rate <HZ>`: Sample rate.
- **Feature**: The hash value of the rendering result is calculated and verified for identity against previous takes.

### `dirty verify <AUDIO_WAV> <CERT_JSON>`
Performs **Acoustic Notarization**. Verifies that the given `.wav` file was indeed generated by the patch state described in the `.dirtyrack.cert` file.
- Checks `render_hash` match.
- Verifies `engine_version` compatibility.
- Reports any `divergence` detected between the certificate and a re-render.

### `dirty bench <PATCH_JSON>`
Runs a performance benchmark on the specified patch. Measures the real-time safety margin of the Gehenna Engine.

### `dirty audit <PATCH_A> <PATCH_B>`
Compares two patch states and outputs a human-readable `diff` of all parameter values.

### `dirty gui`
Launches the main graphical projector (GUI).

## Developer Commands

### `dirty module list`
Displays a list of currently loadable built-in and third-party modules.

### `dirty sdk init <DIR>`
Creates a template project for a new third-party module development.

---

> [!NOTE]
> Currently, DirtyRack focuses on intuitive patching via the GUI, but the deterministic engine equivalent to this CLI always operates behind the scenes to guarantee the reproducibility of all operations.
